# 👩‍💻 Релиз Golang роадмапа

Третий роадмап нашего сообщества - Go.

https://zhukovsd.github.io/golang-backend-learning-course/

Идея такая же, как и в Java / Python роадмапах - инкрементальное нарастание сложности и объёма технологий. Суть проектов не изменилась, но технологии адаптированы под специфику Go.

- [Виселица](https://zhukovsd.github.io/golang-backend-learning-course/projects/hangman/) - практика с базовым синтаксом
- [Симуляция](https://zhukovsd.github.io/golang-backend-learning-course/projects/simulation/) - особенности ООП в Go (в Go нет наследования в таком же виде как в Java и Python)
- [Обменник валют](https://zhukovsd.github.io/golang-backend-learning-course/projects/currency-exchange/) - первый backend проект, REST API без фреймворков
- [Теннисное табло](https://zhukovsd.github.io/golang-backend-learning-course/projects/tennis-scoreboard/) - шаблонизация страниц, unit тесты
- [Погода](https://zhukovsd.github.io/golang-backend-learning-course/projects/weather-viewer/) - интеграция с внешними API
- [Облачное хранилище файлов](https://zhukovsd.github.io/golang-backend-learning-course/projects/cloud-file-storage/) - работа с Minio, Docker, интеграционными тестами через Testcontainers
- [Планировщик задач](https://zhukovsd.github.io/golang-backend-learning-course/projects/task-tracker/) - микросервисы, кафка

Основное идейное отличие Golang бэкенд разработки от Java - отсутствие единого фреймворка, такого как Spring.

За адаптацию роадмапа и ТЗ проектов спасибо Юре [@yosakohf](https://t.me/yosakohf).
